/api/v1/service/ecm/folder/{objectType}/{objectId} get

User must be authenticated and must have 'read' access on the requested object.

get

Get list of objects in the requested container.

get /api/v1/service/ecm/folder/{objectType}/{objectId}

Get list of objects in the requested container.

  • Request
  • Response

URI Parameters

  • objectType: required (string)

    Container type, e.g. COMPLAINT, CASE_FILE, TASK

  • objectId: required (integer)

    Container id

Query Parameters

  • s one of (author,category,created,creator,modified,modifier,name,type,version)

    Sort order

  • dir one of (ASC,DESC)

    Sort direction (ascending or descending)

  • start (integer)

    Paging support - the starting row.

  • n (integer)

    Paging support - the number of rows

  • category one of (Correspondence,Document)

    Used to request only documents, only correspondence, or both; to request both documents and correspondence, do not send a category at all

/api/v1/service/ecm/upload post

post

Add a file. User must have "add file" permission on the parent object. File name must be unique in the folder where the file is added. If the parentCmisFolderId is provided, that folder must be within the parent object's folder tree.

post /api/v1/service/ecm/upload

Add a file. User must have "add file" permission on the parent object. File name must be unique in the folder where the file is added. If the parentCmisFolderId is provided, that folder must be within the parent object's folder tree.

  • Request
  • Response

Body

Type: multipart/form-data

Form Parameters
  • file: required (file)

    Files to be added; file parts can have any name, so long as the MIME type indicates it is a file. All files in the request will be added to the system.

  • category: one of (Document,Correspondence)

    What kind of file is being uploaded.

  • fileType: required (string)

    The logical file type ("other", "roi", ...); not the MIME type.

    Example:

    other, roi, case_file
  • parentObjectType: required (string)

    Type tag for the parent object

    Example:

    CASE_FILE, COMPLAINT, TASK
  • parentObjectId: required (integer)

    Object ID for the parent object

  • parentFolderId: (string)

    Database object ID for the folder to contain the uploaded file. If given, the folder must be within the parent object's folder tree; otherwise an exception is thrown. The uploaded file will be placed in the requested folder. If not given, the folder will be placed in the parent object's root folder.

HTTP status code 200

File successfully added. Body is an array of EcmFile objects.
